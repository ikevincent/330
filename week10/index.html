MDN: Validating Forms

Form validation is the process used to ensure that data entered into forms is in the format required. There are two types of validation: Client-side and server-side. Client-side is done in the browser while server-side is of course done in the server.

There are three main reasons to validate data entered into forms:

  • We want the right data in the right form.
  • We want to protect the user's data.
  • We want to protect our website. (Never trust data passed to your server from the client.)

There are built-in form validation that can be added to form attributes. Required, minlength, maxlength, min, max, type, and pattern.

  • required: specifies whether the field needs to be filled in before it can be submitted.
  • minlength and maxlength: specifies the minimum and maximum length of string data.
  • min and max: specifies the minimum and maximum length of numbers.
  • type: specifies what type the data should be. (Text, email, number, etc.)
  • pattern: specifies a regular expression that defines the pattern expected in a field.
