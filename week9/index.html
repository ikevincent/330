Chapter 9: The Window Object

The window object is the global object of a browser environment. The Browser Object Model (BOM) are the properties and models that contain info about the browser and computer screen.

Any global variable created is property of the window object.

window.alert will pause execution and display a message in a dialogue box.

window.confirm will pause and gives the option of hitting 'ok' (true) or 'cancel' (false).

There are methods to get information about the type of browser, the host of the site, or other bits of information about the browser.

You can access previously visited pages in the browser with window.history.go() which can navigate forward or backwards any number of pages.

Each window object contains a document object.

You can create cookies within the document object by typing something like this: document.cookie = 'name=Superman';

Chapter 14: HTML5 APIs

The data- attribute is a way of embedding data in a web page using custom attributes that are ignored by the browser. They’re also private to a page, so are not intended to be used by an external service – their sole purpose is to be used by a JavaScript program. This means they’re perfect for adding data that can be used as a hook that the program utilizes to access information about a particular element on the page.

The HTML5 specification contains a number of APIs that help to gain access to hardware, such as cameras, batteries, geolocation, and the graphics card.

  • Webstorage - used for storing information locally on the client's devices for easy access during sessions.
  • Geolocation - Find the user's exact location.
  • Web Workers - Adds concurency support to JavaScript.
  • Websockets - Allow two-way communication with a server (Push Messaging).
  • Notifications - Show messages using the system's notifications after requesting access.
  • Multimedia - Easily insert audio and video clips into the html.
