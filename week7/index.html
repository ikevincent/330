Chapter 11: Further Functions

"functions are first-class objects, which means they can be passed around in the same way as every other value. They can have their own properties and methods, as well as accepting other functions as parameters and being returned by other functions."

The call() method can be used to set the value of 'this' inside a function to an object that is provided as the first argument.

You can add custom properties to functions. The example given is a description property to describe what the function does.

IIFE - pronounced 'iffy' is an Immediately Invoked Function Expression

Because they're immediately invoked you have to create the function when you place your () to invoke a function.

By doing this you avoid cluttering the global namespace. Good for when you know you'll only need a short function once.

Event-driven Asynchronous Programming. You can create a callback based on events like running a callback that executes after a certain amount of time.

Spaghetti code is when more than one callback is used in the same function, resulting in a large number of nested blocks that are difficult to comprehend.

This is referred to as "Callback Hell."

Chaper 13: Ajax

Ajax allows web pages to communicate asynchronously with a server. This enables data to be sent and received in the background.

Ajax = Asynchronous JavaScript XML

Fetch API. The fetch method has one argument which is the url of the data you want.

You then handle the code with '.then' and use '.catch' to handle any errors.

When you run the code you check if the response is ok and then throw an error if not with an if block.
