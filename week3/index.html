Object Methods: this

Objects typically represent real world entities like users and orders.

Actions those users make are represented by functions in properties.

"A function that is a property of an object is called its method."

When methods need info that is stored in the object you use "this" followed by a dot, and then the item from within the object.

'this' can be used in any function. It's always evaluated during run-time and it can apply to more than one object.

Chapter 5: Objects

In this chapter, the object discussed are the attributes of Superman but objects can be any collection of relative values.

This is mostly review but these are the important aspects:

  • An Object Literal is just an object created directly in the language.
  • Objects contain key-value pairs.
  • It can also contain methods.
  • You can create objects by creating a const and adding curly braces.
  • If the property key and the variable name it's assigned to are identical you can use a shorthand in ES6 and only write it once.

Use dot notation to call a method from within an object. Like "superman.fly()" to call the fly method from the Superman object.

You can check if properties exist within an object and you can also find all the properties of a object. What is that most commonly used for?

You can nest objects within an object and then drill down in order to retrieve properties of the nested objects.

A namespace helps prevent naming collisions when you reuse variable names. This typically comes up in larger projects or collaborations.

JSON is a string representation of objects and it's lightweight. Use the parse method to return a javaScript object from a JSON string.

Other built-in objects of note:

  • Math - Contains various methods for common mathematic operations
  • Date - Contains info about date and time.
  • RegEx - Searches for strings that match a specified pattern.

Chapter 7: Events

Each time a user interacts with the web page, events happen as they do different things.

Common Event Types:

  • Mouse Events: click, mousedown, mouseup, doubleclick, mouseover, mouseout
  • Keyboard Events: keydown, keyup, keypress
  • Touch Events (on phones): touchstart, touchend, touchenter, touchleave, touchcancel

Using click is sometimes still better on mobile devices because there is a slight delay whereas touch events are instantaneous

You can use preventDefault() to change the default behavior but you have to be careful because users will expect certain things on a web page.
